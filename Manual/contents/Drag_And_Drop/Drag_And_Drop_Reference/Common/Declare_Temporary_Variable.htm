Declare Variável Temporária

Esta ação é usada para declarar e dar um valor inicial a uma variável temporária (também chamada variável local), que é uma variável que só existe pela duração do evento ou script em que foi declarada. Variáveis temporárias são muito úteis para armazenar valores que só são necessários para uma única operação e evitar que seu objects fique "desorganizado" com variáveis que só podem ser usadas uma vez. Também são muito úteis para quebrar operações complexas, ou para definir valores em outras instâncias quando você muda o escopo da ação. Isto porque as variáveis declaradas como arenotscopiáveis temporárias para qualquer instância em particular, mas em vez disso, escopadas para o evento ou para o próprio script, portanto, se você criar uma variável temporária e depois alterar o escopo de ação usando a ação Aplicar a..., você ainda pode verificar a variável temporária sem problemas e sem alterar seu escopo de ação.

Vale notar que quando falamos de "valores", não nos referimos apenas a valores numéricos, pois uma variável pode ser um string, um ponteiro, um ID de recurso ou qualquer outra coisa que uma função possa retornar ou usar, até mesmo uma expressão. Veja a seção sobre tipos de dados para mais informações.

Note que muitas ações têm uma caixa de seleção "Temp" que permite criar uma nova variável local temporária para manter o valor de retorno da ação, e esta variável pode ser acessada na ação script ou evento como qualquer outra variável depois disso. Observe também que as variáveis locais temporárias são escopadas para o script ou evento no qual elas são criadas, e assim elas podem ser usadas por outras instâncias no mesmo bloco ao usar o Aplicar a.

Você também pode adicionar variáveis adicionais na mesma ação clicando no ícone de mais ao lado da ação, e dando outra variável local temporária nome e valor. Para informações mais avançadas sobre variáveis, consulte a seção sobre Variáveis e Escopo de Variáveis.
