Construindo o Código de Bloco de Ação

Para adicionar comportamentos a objects você pode construir seu código usando as Ações das diferentes bibliotecas disponíveis na Caixa de Ferramentas de Ação. Para começar, você precisará fazer um novo projeto visual GML e depois fazer um novo objeto (você pode atribuir um sprite para o object também, se necessário). Em seu novo object, você pode começar a adicionar eventse nos eventos acrescente suas ações de código visual GML.

Observe que quando você adiciona um novo evento, uma janela de "código" é aberta com uma aba para o evento em questão (veja a imagem acima), e agora você pode arrastar qualquer ação que desejar da Caixa de ferramentas à direita para o painel de blocos de ação da janela de código. Agora, embora seja verdade que você pode adicionar qualquer ação, isso não significa que todas elas funcionarão ou que o projeto irá compilar com elas. Algumas ações requerem pelo menos uma variável para funcionar, enquanto outras - como as ações de Desenho - só funcionarão se usadas em um evento específico. Como você sabe quais utilizar? Bem, geralmente é simplesmente uma questão de usar a lógica... se uma ação requer uma variável e ainda não a definimos, então não devemos usá-la até que tenhamos adicionado uma ação para criar a variável.

Quando você arrasta uma ação da Caixa de Ferramentas para o Bloco de Ação principal workspace, ela se expandirá para mostrar osargumentos disponíveis (parâmetros) que você pode preencher e mudar para definir o comportamento. Variáveis-alvo

É importante notar que muitas ações oferecem uma variável "alvo", que pode ser sinalizada como "Temporária", por exemplo:

O que isto significa é que você pode fornecer uma variável que é o alvo para manter o valor retornado pela ação. No exemplo de ação acima, a ação retornará o volume do recurso de áudio que você selecionar, então você fornece uma variável alvo para manter este valor para que você possa referenciá-lo mais tarde.

Agora, esta variável alvo precisará ter sido declarada anteriormente ou usando Assign Variable (que cria uma Variável de Instância) ou Declare Temporary Variable (que cria uma Variável Temporária Local). Se você marcar a opção "Temp", então você pode simplesmente adicionar um nome de variável e a ação criará aquela variável e a definirá para manter o valor de retorno para você (criando uma variável local temporária). Qualquer ação usada depois disto pode agora acessar o valor na variável temporária, mas somente dentro do mesmo Evento ou Script. As variáveis temporárias estão disponíveis apenas dentro do escopo em que foram criadas. Para informações mais detalhadas sobre variáveis e escopo de variáveis, por favor, veja aqui.

Auto Completo

Ao trabalhar com GML Visual você terá que adicionar variáveis e expressões nos diferentes campos de entrada das ações. No entanto, ao fazer isso, muitas vezes a janela Auto Completo aparecerá para ajudá-lo:

Esta janela pop up listará todas as variáveis, constantes e funções construídas em GML (GameMaker Language), assim como quaisquer recursos que contenham as letras iniciais do que você estava digitando. Ela pode ser usada para encontrar rapidamente o recurso ou variável que você deseja para a ação sem ter que digitar tudo em você mesmo. Por exemplo, se todo o seu rooms tiver o prefixo "rm_", então digitando isso e aguardando um momento aparecerá a janela autocompleta com todo o assets começando com "rm_". Note que todas as variáveis embutidas que são mostradas na janela auto complete podem ser usadas em qualquer lugar onde uma variável ou expressão é esperada em uma ação, assim como a maioria das funções do GML.
