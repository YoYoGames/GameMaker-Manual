GMRT Preferences

This section contains preferences for GMRT (GameMaker Runtime):

  • Path to GMRT: This should be where the GMRT packages are installed.
  • Path to ...: The rest of the path options are optional and provided in case you want to use your own tools for GMRT compilation. GMRT packages should already include all of this and you will not need to change these options unless you specifically want to.
    • Path to the GMRT Asset Compiler: Path to a custom asset compiler for compiling your game WADs.
    • Path to Clang executable: Path to your own installed Clang binary. Also accepts executable name if executable directory is in PATH.
    • Path to Windows SDK + VC Tools root: Path to your own Windows SDK/VC Tools root directory.
    • Path to Windows SDK: Path to only the Windows SDK, previous combined option overrides this if set.
    • Path to VC Tools: Path to only the VC Tools root, previous combined option overrides this if set.
    • Path to CMake executable: Path to custom CMake installation. Also accepts executable name if executable directory is in PATH.
    • CMake generator: Specifies which CMake generator should be used when building a project. By default this is set to Ninja, but can be changed to "Visual Studio 17" to build a VS 2022 project.
    • Path to CMake generator: Path to a custom CMake generator if a non-default CMake Generator is needed.
  • Compile with verbose output: This enables detailed logging for the compiler output which can help debug issues when submitting reports. This may slow down compilation.
  • Build Type: The build of GMRT to use.
