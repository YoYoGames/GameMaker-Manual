debug_event

Esta função gera um evento de depuração personalizado que será mostrado na visualização gráfica do debugger quando um jogo estiver sendo executado no modo de depuração. Se você precisar que mensagens sejam exibidas quando não estiver no modo de debug, use show_debug_message().

A função também terá duas reservadas strings para ajudar a realizar debugging utilizando aplicações externas como Visual Studio. Estas strings são:

  • "OutputDebugOn" - Isto permite uma chamada para OutputDebugString para o alvo do Windows, o que significa que toda a saída de depuração - tudo que você vê na janela de saída - pode ser visualizado por Visual Studio ou por aplicativos de terceiros.
  • "OutputDebugOff" - Desativa o comportamento descrito acima.
  • "BreakOnError" - Esta opção é apenas para Windows YYC builds, e significa que os projetos não exibirão a tela habitual de erro de código se o runtime detectar um erro, mas, em vez disso, apenas continuar e travar. Isto permite que você veja o stacktrace dentro do Visual Studio se debugging.
  • "ResourceCounts" - Lista todos os recursos atualmente ativos, tais como Estruturas de Dados, Fontes de Tempo, Superfícies, etc.
  • "DumpMemory" - Fornece informações sobre o uso atual da memória.

Sintaxe:

debug_event(string)

Argumento Tipo Descrição
string Cordão The custom debug event string to use.

Devoluções:

N/A

Exemplo:

if !surface_exists(global.EffectsSurface)
{
    debug_event("Recreating Effects Surface");
    global.EffectsSurface = surface_create(room_width, room_height);
}

O código acima verifica se uma superfície existe e se não existe, um evento de depuração é acionado na visualização gráfica do debugger (o jogo deve ter sido executado no modo de depuração para que isso seja visível) e a superfície é recriada.
