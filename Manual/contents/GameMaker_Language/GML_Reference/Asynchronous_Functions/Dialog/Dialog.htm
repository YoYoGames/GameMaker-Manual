Diálogo

A maioria dos computadores e dispositivos móveis modernos não gostam quando um jogo é parado enquanto uma janela de diálogo é mostrada. Eles tendem a interpretar isto como um corredor de jogo tendo parado, e podem considerá-lo um erro e fechar o aplicativo prematuramente. Para evitar que este GameMaker tenha funções assíncronas de diálogo que permitem solicitar certas formas de feedback ou entrada do usuário, mas sem bloquear o game runner a qualquer momento, significando que - quando chamado - o jogo será executado normalmente em segundo plano enquanto o usuário responde à janela de diálogo mostrada. Uma vez que o usuário tenha respondido, será chamado um Evento Assíncrono de Diálogo para que você possa lidar com os resultados.

NOTA: Com exceção de shop_leave_rating()Estas funções são apenas para depuração e testee não devem ser utilizadas em jogos lançados. Para isso, você deve criar uma interface de usuário personalizada para receber informações dos jogadores, para que você tenha controle total sobre a aparência e o comportamento dos diálogos.

Abaixo está uma lista de todas as funções de diálogo assíncrono disponíveis:

  • get_login_async
  • get_string_async
  • get_integer_async
  • show_message_async
  • show_question_async
  • shop_leave_rating
