http_set_connect_timeout

This function sets the connection timeout used for HTTP requests. Any requests made after a call to this function will use this timeout.

When the connection times out, an async HTTP event will be triggered.

The default timeout value is 60000 ms, i.e. one minute.

 The value set is only a guideline and requests may time out earlier or later depending on the network conditions and the operating system.

 

Syntax:

http_set_connect_timeout(connect_timeout_ms);

Argument Type Description
connect_timeout_ms Real The connection timeout value to use for subsequent requests (in milliseconds)

 

Returns:

Real

 

Example:

Mouse Left Pressed Event

http_set_connect_timeout(10000);
var _map_headers = ds_map_create();
request_id = http_request("https://www.google.com", "GET", _map_headers, "");

Async HTTP Event

if (async_load[? "id"] != request_id)
{
    show_debug_message("Not a response to our request...");
    exit;
}

var _status = async_load[? "status"];
var _http_status = async_load[? "http_status"];
show_debug_message($"Status: {_status}, HTTP Status: {_http_status}");

The above code first sends an HTTP GET request in the Mouse Left Pressed event. Before the request, the HTTP timeout is first set to 10 seconds. In the async HTTP event the async ID is compared to the request ID returned by the call to http_request. If the IDs are not the same, a debug message is shown and the event is exited. If the async ID equals the request ID, code execution continues and the values of the async_load's "status" and "http_status" keys are shown in another debug message. The values of these variables should be checked to determine if an error occurred and, if so, which one.
