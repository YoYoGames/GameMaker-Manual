Funções Numéricas

Números reais no GameMaker são considerados números de ponto flutuante de precisão dupla - ou seja, eles têm um ponto decimal sem número fixo de dígitos antes ou depois do ponto - ou inteiros - ou seja, são números inteiros sem valor de ponto decimal. 2, por exemplo, é um número inteiro, mas 2,01 é um ponto flutuante real.

NOTA: No alvo HTML5, todos os números reais são duplas.

Esta distinção entre inteiros e flutuadores é muito importante quando se trata do desenvolvimento de plataformas cruzadas, pois a precisão dos cálculos feitos em um Windows PC não é a mesma que a precisão desses mesmos cálculos quando feitos em um dispositivo móvel. Isto significa que você deve prestar atenção especial ao fazer comparações, por exemplo:

if (image_index == 1)
{
    do something...
}

No exemplo acima, se estivermos definindo o image_speed como 0,25 - por exemplo - então após quatro passos você pode assumir que o valor image_index seria 1, mas pode ser um valor como 1,0000002 devido à forma como a matemática de ponto flutuante funciona e, portanto, a avaliação não será true. Estes tipos de erros podem ser bastante difíceis de depurar e por isso é sempre bom estar atento a eles e planejar com antecedência e usar outros meios de verificação de valores ou usar as funções apropriadas de piso ou arredondamento (listadas abaixo) para converter o número para verificar em um número inteiro (para mais informações sobre matemática de ponto flutuante e porque isto é um problema, por favor veja aqui). Por exemplo, o código acima poderia ser escrito como:

if (floor(image_index) == 1)
{
    do something...
}

Também vale a pena notar que ao utilizar os alvos do Compilador YoYo, todas as expressões e funções são avaliadas da esquerda para a direita, enquanto em todas as outras plataformas alvo são avaliadas da direita para a esquerda, o que significa que isto - por exemplo - dará resultados diferentes dependendo da plataforma:

val = max(num, ++num, num++);

NOTA: Para mais informações, consulte a seção sobre Ordem de Avaliação.

Você também pode usar uma função especial disponível no GameMaker para definir o valor epsilon para matemática de ponto flutuante. Quando um número real é arredondado para o número de ponto flutuante mais próximo, o epsilon (também conhecido como "epsilon máquina") forma um limite superior no erro relativo, e você pode obter e definir o valor do epsilon usando as seguintes funções:

  • math_set_epsilon
  • math_get_epsilon
