draw_primitive_begin

This function starts drawing a primitive of the given type (this must be called before you can define any primitives).

There are 6 types of primitives you can define as the following constants:

The following image illustrates basically how these should look and also the order in which you should define the vertices:

 On some platforms (Windows, Xbox) the pr_trianglefan type is not natively supported and so GameMaker does a conversion when the game is compiled to make them work. This means that on those platforms the pr_trianglefan type will be much slower to use than the others.

 

Syntax:

draw_primitive_begin(kind);

Argument Type Description
kind Primitive Type Constant The kind of primitive you are going to draw

 

Returns:

N/A

 

Example:

var _steps = 20;
var _xx = 50;
var _yy = 50;
var _radius = 30;
draw_primitive_begin(pr_trianglefan);
draw_vertex(_xx, _yy);
for(var i = 0; i <= _steps; ++i)
{
    draw_vertex(_xx + lengthdir_x(_radius, 270 * i / _steps), _yy + lengthdir_y(_radius, 270 * i / _steps));
}
draw_primitive_end();

The above code will draw three quarters of a circle made from primitives.
