video_open

This function opens a video file using the path specified. The path can refer to a file in the Included Files of your project, a file locally present on the user's device, a URL for loading a file from the internet (which may not work on all platforms), or a constraints object to display a camera feed as a video (only supported on GX.games).

As you can only load one video at a time, this function loads the specified video into the only available "video slot", which is then manipulated via the other Video functions.

When passing a constraints object to show a video feed on GX.games you can also enable microphone audio.

There are asynchronous callbacks associated with video playback. See Async Callbacks for more information.

 

Syntax:

video_open(path);

Argument Type Description
path String Path to the video file or a constraints object for a camera feed on GX.games

 

Returns:

N/A

 

Example: Basic Use

video_open("splash.mp4");

The code above loads splash.mp4 from the Included Files of the game.

 

Example 2: Camera Feed on GX.games

var _constraints = 
{
    audio: true,
    video: true
};

video_open(json_stringify(_constraints));

The code above requests access to the device's built-in camera and microphone and attempts to open the camera feed as a video. A constraints object is passed to video_open in which permission for both audio and video is requested without any specific requirements.
