xboxlive_stats_get_leaderboard

Esta função pode ser usada para recuperar uma tabela de classificação global para uma determinada estatística. Você fornece o ID do usuário (como retornado, por exemplo, pela função xboxlive_get_user()), o stat string (como definido quando você o registrou como "Featured Stat"), e então você especifica uma série de detalhes sobre quais informações do leaderboard você quer recuperar. Observe que você só pode recuperar um quadro de líderes global para int ou estatísticas reais, mas não para string estatísticas.

IMPORTANTE! As estatísticas utilizadas nas placas de líderes globais devem ser registradas como "Estatísticas em Destaque" no XDP/Windows Dev Center, caso contrário, um erro será devolvido. Se você deseja quadros de líderes locais (sociais), por favor veja a função xboxlive_stats_get_social_leaderboard().

A função gerará uma chamada de retorno que desencadeará um Evento Assíncrono Social. Este evento terá o mapa DS integrado. async_load que deve então ser analisado para as seguintes chaves:

  • "id" - Manterá a constante achievement_stat_event
  • "evento" - Realizará o string "GetLeaderboardComplete".
  • "userid" - O ID do usuário associado à solicitação
  • "erro" - 0 se bem sucedido, algum outro valor se tiver havido um erro. Os dois seguintes são os erros mais comuns devolvidos:
    • 2145844844: Não é possível encontrar o quadro de líderes solicitado (o estatuto não está registrado como um estatuto em destaque)
    • -2145844848: Má solicitação (o estatuto não é um tipo de quadro de líderes válido, deve ser um string)
  • "errormessage" - A string com uma mensagem de erro, se houver alguma disponível
  • "display_name" - A identificação única para o painel de controle, conforme definido no painel do fornecedor.
  • "numentries" - O número de entradas no quadro de líderes que você recebeu.

O restante do mapa DS também conterá os dados do quadro de líderes com o seguinte formato (onde "N" é a posição nos dados do quadro de líderes, de 0 a "numentries"):

  • "PlayerN" - O nome do jogador, onde "N" é um valor inteiro correspondente à sua posição dentro da lista de entradas do quadro de líderes.
  • "PlayeridN" - A identificação única do usuário do jogador, "N".
  • "RankN" - A classificação do jogador "N" dentro da tabela de liderança.
  • Pontuação "N"- A pontuação do jogador "N".

Sintaxe:

xboxlive_stats_get_leaderboard(user_id, stat, num_entries, start_rank, start_at_user, ascending);

Argumento Tipo Descrição
user_id ID do usuário do Xbox The user ID of the user to get the leaderboard for
stat Cordão The stat (as a string) to create the global leaderboard from
num_entries Real The number of entries from the global leaderboard to retrieve
start_rank Real The rank in the leaderboard to start from (use 0 if the "start_at_user" argument is set to true)
start_at_user Booleano Set to true para começar no posto de identificação do usuário, false caso contrário (ajuste para false se o argumento "start_rank" for qualquer coisa diferente de 0)
ascending Booleano Set to true para ordem ascendente e false para ordem descendente

Exemplo ampliado:

A seguir, um exemplo ampliado de como esta função pode ser utilizada. Para começar, você a chamaria em algum evento como Room Start ou Create:

xboxlive_stats_get_leaderboard(user_id, "GlobalTime", 20, 1, false, true);

O código acima seria chamado para obter uma lista de todas as posições de liderança social para o jogo, e geraria uma chamada de retorno de Evento Assíncrono Social, que trataríamos da seguinte forma:

if (async_load[? "id"] == achievement_stat_event)
{
    if (async_load[? "event"] == "GetLeaderboardComplete")
    {
        global.numentries = async_load[? "numentries"];
        for(var i = 0; i < numentries; i++;)
        {
            global.playername[i] = async_load[? "Player" + string(i)];
            global.playerid[i] = async_load[? "Playerid" + string(i)];
            global.playerrank[i] = async_load[? "Rank" + string(i)];
            global.playerscore[i] = async_load[? "Score" + string(i)];
        }
    }
}

O código acima verifica o mapa DS retornado no Evento Assíncrono Social e se seu "id" corresponde à constante a ser verificada, ele então verifica se o evento foi acionado pelos dados do quadro de líderes retornados antes de fazer looping pelo mapa e armazenar todos os diferentes valores em uma série de arrays globais.
