display_get_dpi_x

This function gets the DPI of the device display along the x axis (this value is also dependent on the orientation of the device).

Dots per inch (DPI) is a measure of spatial printing or video dot density, in particular the number of individual dots that can be placed in a line within the span of 1 inch (2.54 cm). When working on mobile devices (in particular Android devices) this is an important factor to take into consideration as what may be appropriate for one display resolution, may not be appropriate for another one. For example, you may have two displays with the same resolution of 400 x 800, but display 1 has a DPI of 60 and display 2 has a DPI of 30. In this case, any text or image displayed on display 2 will appear much larger, even though the actual resolution is the same.

 Mac and iOS do not return specific DPI settings but appear to return the same values as the OS, which are not correct (but will have to do) as Apple do not give the correct values.

 

Syntax:

display_get_dpi_x()

 

Returns:

Real

 

Example:

dpx = display_get_dpi_x();

This would set the variable dpx to the DPI value of the x axis.
