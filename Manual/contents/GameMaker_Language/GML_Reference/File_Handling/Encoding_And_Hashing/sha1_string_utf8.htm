sha1_string_utf8

Na criptografia, SHA-1 é uma função de hash criptográfica projetada pela Agência de Segurança Nacional dos Estados Unidos e é empregada em várias aplicações e protocolos amplamente utilizados, como o popular Git, onde é usada para verificar alterações de arquivos. Esta função irá pegar uma entrada UTF-8 string (que tem um número variável de bytes por caractere) e retorna uma digestão de mensagem de 160 bits em formato ASCII, por exemplo:

sha1 input string = "The quick brown fox jumps over the lazy dog"
sha1 output hash = "2fd4e1c6 7a2d28fc ed849ee1 bb76e739 1b93eb12"

Desta forma, é possível gerar uma chave segura que pode ser armazenada e utilizada para verificar a integridade das informações enviadas (ou recebidas de) um server externo (por exemplo).

NOTA: Existem dois formatos para a codificação sha1, UTF-8 e unicode. Ambos são fornecidos para facilitar a comunicação com diferentes configurações server, mas o mais comum a ser usado é unicode.

NOTA: O SHA-1 não é completamente seguro e pode ser quebrado. Veja esta página para mais informações.

Sintaxe:

sha1_string_utf8(string)

ArgumentoTipo Descrição
string The string to hash.

Devoluções:

Exemplo:

var hash, str;
str = json_encode(hiscore_map);
hash = sha1_string_utf8(str);
ini_open("local.ini");
ini_write_string("info", "0", hash);
ini_close();
get[0] = http_post_string("http://www.MacSweeney Games.com/CatchTheHaggis?game_hiscores=" + string(global.game_id), str)

O código acima codificará um mapa DS em um JSON string. Um hash sha1 é então gerado e armazenado em um arquivo ini para que este possa ser usado posteriormente para verificar a integridade do JSON caso as mesmas informações sejam recebidas posteriormente do JSON server. O JSON é então enviado.
