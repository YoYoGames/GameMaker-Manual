Camadas do mapa de azulejos

O Editor do GameMaker Room permite adicionar tiles em qualquer room usando camadas. Como o nome indica, uma camada do mapa tile é simplesmente uma coleção de tile set tiles que são todos considerados na mesma profundidade dentro do room, e empilhando camadas você pode fazer com que algumas coisas se sobreponham a outras ou se subponham a outras. Você também pode controlar certos aspectos das camadas a partir do código, adicionando ou removendo coisas, ou alterando certas propriedades da camada ou o que ela contém em tempo de execução. Observe que no editor room você está limitado a um mapa tile por camada (o Camada de azulejos), mas ao trabalhar com eles em código, você pode ter vários mapas tile atribuídos a uma única camada, com cada mapa tendo sua própria identificação e propriedades únicas, portanto:

  • O conjunto de azulejos é o recurso que você utiliza para criar tile camadas e tile mapas
  • A camada de azulejos é o que você cria no Editor Room para adicionar tile sets a
  • O mapa de azulejos é o que você chama de coleção de tiles que são adicionados a uma camada, seja no editor room ou através de código, como um único elemento sobre essa camada

Além das funções listadas acima para definir mapas tile, você também tem uma série de funções que podem ser usadas para manipular os dados de cada indivíduo tiles. Um "blob" de dados tile é composto do índice tile mais alguns bits extras para mostrar se o tile foi girado, invertido ou espelhado. Você também tem a possibilidade de adicionar sua própria máscara de bits aos dados tile, embora isto seja apenas para usuários avançados, pois você estará mascarando os bits reservados para o índice tile - essencialmente há 19 bits reservados para seus índices tile, mas se você usar apenas 8 deles, então você tem onze bits restantes que podem ser usados para criar uma máscara personalizada. Esta máscara pode então ser usada e verificada em código para criar colisões personalizadas tile ou o que quer que seja. Entretanto, você normalmente não precisaria criar seus próprios blobs de dados tile, e em vez disso usaria a função tilemap_get() para obter os dados tile, então você os manipularia usando as funções abaixo, e então você os definiria novamente usando tilemap_set().
