font_add_sprite

Com esta função, você pode usar uma "faixa sprite" para criar uma nova font asset, onde cada subimagem seria um símbolo ou letra individual. O índice é o número do mapa UTF8 para o primeiro caractere a ser usado e as faixas a serem usadas são as mesmas que você usaria para um normal font, começando com "!", mas lembre-se que você só deve incluir o que você precisa e que você deve estar usando o ord() para obter o valor UTF8 correto para a letra inicial. O criado sprite font terá então as subimagens definidas para um glifo individual do font, onde o "primeiro" glifo terá a imagem_index de 0. Você pode definir "espaço" como sendo qualquer caractere que você queira, por exemplo, uma única linha do tamanho que você quer que o espaço seja, e todos os espaços no texto serão renderizados nessa largura (mas a imagem escolhida nunca será renderizada), ou se você não fornecer um sprite para espaço, então a largura do caractere mais largo no sprite font será usada em seu lugar.

Você também pode definir o espaçamento para que o font seja proporcional (true) ou não (false), onde um font proporcional é espaçado com base na largura real das letras (assim a letra "i" leva menos room do que a letra "w", por exemplo), enquanto um font não proporcional será espaçado com base na largura da subimagem, como um monospaced font. Finalmente, você pode definir o espaço a ser deixado entre cada letra ao escrever, e este pode ser qualquer valor inteiro, sendo que 0 não tem espaço (as letras "tocarão" se forem proporcionais).

A função retorna um valor de índice que deve ser armazenado em uma variável, pois isso será necessário em todos os outros códigos que se referem a este font, ou retornará -1 se a função falhar por qualquer razão.

Vale notar também que não pode haver espaços no mapa de caracteres, de modo que se você, por exemplo, usar '.' e '0' mas não '/', você precisa incluir '/' de qualquer forma ou deixar uma subimagem em branco entre os dois caracteres usados.

O próprio sprite pode ser um sprite asset do navegador Asset ou um carregado de um arquivo usando sprite_add primeiro para obter o índice do sprite carregado (que depois é usado na função). É importante notar que para as funções de alinhamento do font (como draw_set_halign()) para funcionar como esperado ao utilizar um sprite font, o sprite deve ter sua origem definida para o canto superior esquerdo. Se você usar outros valores, então você precisará levar em consideração a compensação de origem ao desenhar o texto usando o font.

NOTA Quando você carrega um font no GameMaker, você deve se lembrar de removê-lo novamente (com font_delete()) quando não for mais necessário, caso contrário existe o risco de um vazamento de memória que irá retardar e, eventualmente, quebrar seu jogo.

Sintaxe:

font_add_sprite(spr, first, prop, sep);

Argumento Tipo Descrição
spr Ativo Sprite The sprite to add a font based on.
first Real The index of the first available character.
prop Booleano Set as proportional font or not.
sep Real The space to leave between each letter.

Devoluções:

Font Asset

Exemplo:

global.Font = font_add_sprite(spr_HUD_Font, ord("!"), true, 2);

O código acima criará um novo font asset a partir de um sprite asset e armazenará o índice do novo font na variável "global.Font" para uso posterior.
