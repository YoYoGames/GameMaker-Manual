Gerando Object Eventos

OGameMaker é completamente guiado por eventos, e que todas as ações acontecem como resultado de certos eventos que são acionados em uma ordem específica ou por uma ocorrência específica dentro do jogo. Há uma série de eventos diferentes, cada um com uma tarefa ou momento específico e alguns deles são mesmo divididos em eventos "sub" separados. Normalmente estes eventos são executados pelo GameMaker quando ele detecta um gatilho dentro do jogo, ou cada tick do jogo (como é o caso dos eventos de etapa e sorteio), mas você também pode aplicar um evento a uma instância no atual room de dentro de um pedaço de código.

Existem as seguintes funções que lidam com a geração de eventos:

  • event_inherited
  • event_perform
  • event_perform_async
  • event_perform_object
  • event_user

Há também uma série de variáveis que podem ser usadas apenas para leitura com as funções listadas acima e que são particularmente úteis para depuração:

  • event_action
  • event_number
  • event_object
  • event_type

Para mais informações sobre eventos, consulte a seção Eventos Objeto.
