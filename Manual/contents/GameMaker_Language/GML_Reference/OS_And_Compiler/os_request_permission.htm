os_check_permission

Com esta função, você pode solicitar uma permissão específica. Você fornece a permissão para solicitar como um string usando o formato "android.permission.<permission>", assim, para solicitar a permissão RECORD_AUDIO (por exemplo) você chamaria

os_request_permission("android.permission.RECORD_AUDIO");

A função irá acionar um Evento Assíncrono do Sistema onde o async_load O mapa DS conterá os seguintes pares chave/valor:

  • "tipo": Este é o tipo de evento que foi acionado e o valor será o string "permission_request_result".
  • "<permissão>": Esta será a permissão solicitada pela função como um string, que conterá uma das seguintes constantes:
Constante Descrição
os_permission_granted Isto indica que a permissão foi concedida
os_permission_denied Isto indica que a permissão não foi concedida

Vale notar que as seguintes permissões são apoiadas nativamente pelo GameMaker, mas são consideradas "perigosas" pelo Google e como tal devem ser explicitamente solicitadas (note também que algumas permissões podem ser solicitadas usando as Opções de Jogo Android sem a necessidade desta função):

  • android.permission.WRITE_EXTERNAL_STORAGE
  • android.permission.READ_PHONE_STATE
  • android.permission.RECORD_AUDIO

Para mais informações sobre permissões de aplicativos, consulte a Documentação do Android.

IMPORTANTE! Esta função é apenas para o alvo Android.

Sintaxe:

os_request_permission(permission)

Argumento Tipo Descrição
permission Cordão The permission to request (a string)

Devoluções:

N/A

Exemplo:

if (os_type == os_android)
{
    if (os_check_permission("android.permission.INTERNET") == os_permission_denied)
    {
        os_request_permission("android.permission.INTERNET");
    }
}

O código acima verifica o tipo de sistema operacional e se for Android, realiza uma verificação das permissões e se a permissão "INTERNET" ainda não tiver sido concedida, ela a solicita.
