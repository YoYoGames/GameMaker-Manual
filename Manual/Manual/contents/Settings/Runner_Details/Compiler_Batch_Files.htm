Compilador de Arquivos em Lote / Scripts

O compilador GameMaker permite que você coloque arquivos Batch (Windows) ou Shell scripts (em macOS e Linux) no diretório de seu projeto, e no diretório para cada extension em seu projeto.

Colocação

Um arquivo script pode ser colocado na raiz do diretório de seu projeto, junto com o arquivo .yyp do projeto.

Um arquivo script também pode ser colocado no diretório de um extension, que pode parecer assim: <project_directory>/extensions/<extension_folder>/.

As versões extension de um arquivo script serão executadas primeiro, e sua versão raiz será executada por último. A ordem na qual extensions é executada não pode ser controlada, portanto um desenvolvedor extension deve garantir que uma extensão scripts não confie em outra scripts.

Execução

Arquivos de script com nomes específicos, colocados em qualquer um dos diretórios descritos acima, são executados em vários pontos durante o processo de compilação:

Nome do arquivo do Windows Batch macOS/Linux Shell Script Nome Descrição
pre_run_step.bat pre_run_step.sh Isto é executado antes que o jogo esteja prestes a começar.
post_run_run_step.bat post_run_step.sh Isto é executado quando o jogo está preparado e pronto para ser executado; após a execução do roteiro, o jogo é iniciado
pre_build_step.bat pre_build_step.sh Isto é executado antes que o compilador asset seja iniciado para a construção do jogo.
post_build_step.bat post_build_step.sh Isto é executado após o compilador asset ter sido iniciado para a construção do jogo
pre_package_step.bat pre_package_step.sh Isso é executado antes da etapa final de embalagem, que é quando todos os arquivos estão prontos, mas o arquivo ZIP final ou a embalagem da loja está prestes a ser criada.
post_package_step.bat post_package_step.sh Isto é executado após a etapa final de embalagem ter sido concluída e o arquivo ZIP final ou a embalagem da loja ter sido preparada
post_textures.bat post_textures.sh Isto é executado após o textures para seu jogo ter sido gerado. Você pode usar esta etapa para otimizar seu jogo textures manualmente.

Este script recebe uma variável de ambiente chamada TexturesDir, que armazena o path para o gerado textures.

Estes arquivos script são suportados ao construir em todas as plataformas alvo.

Desabilitando Scripts para uma extensão

Se você utiliza múltiplos extensions que rodam scripts, por exemplo, Steamworks e GDK, então você vai querer desativar a Steamworks ao exportar para a GDK, e vice-versa.

Para isso, veja: Como desativar Extensions?
