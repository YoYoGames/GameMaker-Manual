Se o botão Gamepad for liberado

Esta ação verifica quando um botão gamepad foi liberado para o índice gamepad dado. Ele retornará true no único tique do jogo em que o botão é liberado e false em todos os outros momentos. Se você ativar o modificador not, esta ação será revertida, e verifique se o botão não foi lançado, retornando false no tique do jogo em que foi lançado e true em todos os outros momentos.

Se o botão a ser verificado for um botão analógico, então a liberação não será detectada até que o valor do botão tenha ficado abaixo do limite definido usando a ação Set Gamepad Button Threshold.

Note que para adicionar ações no bloco "se", elas devem ser descartadas ao lado da ação, como mostrado na imagem abaixo:

Essas ações serão executadas agora se o "se" avaliar true enquanto quaisquer ações descartadas em outro lugar serão executadas após o bloco "se".

Argumentos:

Argumento Descrição
Gamepad O índice gamepad.
Button O nome do botão a ser verificado.
Not Negar o cheque (o verdadeiro torna-se falso e vice-versa)

Exemplo:

O código de bloco de ação acima pesquisa o estado do botão do gamepad 0 a cada passo e se um botão estiver sendo mantido pressionado, ele verifica se o botão inicial está pressionado. Se a verificação for true na pressão inicial para baixo, um som é tocado, então, enquanto o botão é mantido pressionado para baixo, a velocidade da instância é ajustada para 10. Se o botão não estiver sendo pressionado, a velocidade é ajustada para 10. mantido pressionado, uma verificação é feita no botão de liberação para repor a velocidade em 0 novamente.
