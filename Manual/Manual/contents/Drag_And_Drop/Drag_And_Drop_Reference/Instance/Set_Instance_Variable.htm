Variável de instância fixa

Com esta ação, você pode definir uma das variáveis embutidas. Todas as instâncias de todas objects têm uma série de variáveis já criadas dentro delas para conter informações essenciais como a posição x/y atual, a velocidade, a direção, a visibilidade e muitas outras coisas. As variáveis de instância que você pode usar são:

  • X Coordenada - A posição ao longo do eixo x no room.
  • Coordenada Y - A posição ao longo do eixo y no room.
  • Direção - A direção de movimento para a instância (o padrão é 0°, onde a direita é 0°, para cima é 90°, esquerda é 180° e para baixo é 270°).
  • Velocidade - A velocidade (em pixels por frame de jogo) em que a instância está se movendo.
  • Velocidade horizontal - A velocidade horizontal (em pixels por frame de jogo) em que a instância está se movendo. Uma velocidade negativa significa que ela está se movendo esquerda, uma velocidade positiva é a direita e o padrão é 0.
  • Velocidade vertical - A velocidade vertical (em pixels por frame de jogo) em que a instância está se movendo. Uma velocidade negativa significa que ela está se movendo para cima, uma velocidade positiva é baixa e o padrão é 0.
  • Visível - A variável de alternância de visibilidade (padrão é true, a menos que definido de outra forma no Editor de Objetos, programado para false para torná-lo invisível). Note que uma instância invisível ainda executa todos os seus eventos , exceto o Evento de Sorteio.
  • Persistente - A variável de alternância de persistência (padrão é false, a menos que definido de outra forma no Editor de Objetos, programado para true para fazer persistente). Isto fará com que uma instância "persistente" de room a room, ou seja: a instância não será limpa no final de room, mas será transportada para o próximo room. Note que quando uma instância é persistente não chamará seu evento Create nor Destroy, mas chamará o Clean Up, Room End, e Room Start events.
  • Camada - A camada para a qual a instância foi designada.
  • Profundidade - O valor de profundidade atribuído à camada.
  • Sprite - O recurso sprite designado para a instância.
  • Índice de imagem - O índice de imagem do sprite atribuído. Este valor define o quadro do sprite atribuído à instância, e se o valor é maior do que o número de quadros na animação que "loop" de tal forma que um sprite com 5 quadros, se ajustado ao quadro 7, mostrará realmente o quadro 2.
  • Ângulo de imagem - O ângulo em que a instância deve ser definida (o padrão é 0°, onde a direita é 0°, acima é 90°, esquerda é 180° e abaixo é de 270°).
  • Imagem Alfa - O valor alfa para a instância (padrão é 1) de 0 a 1, onde 0 é totalmente transparente e 1 é totalmente opaco.
  • Image Blend - A cor a ser misturada com a instância sprite (o padrão é branco).
  • Velocidade da imagem - A velocidade da animação, por exemplo.
  • Escala Horizontal - A escala horizontal na qual a instância sprite deve ser desenhada (isto também afetará a instância delimitadora caixa e colisões). Uma escala de 1 indica que não há escala (1:1), valores menores serão escalados para baixo (0,5, por exemplo, metade da largura da sprite), valores maiores serão escalados para cima e valores negativos serão espelhados no sprite e escalados a menos que o valor utilizado é exatamente -1 (nesse caso o sprite é apenas espelhado ao longo do eixo y, sem escala).
  • Escala Vertical - A escala vertical na qual a instância sprite deve ser desenhada (isto também afetará a instância delimitadora caixa e colisões). Uma escala de 1 indica que não há escala (1:1), valores menores serão escalados para baixo (0,5, por exemplo, metade da altura do sprite), valores maiores serão escalados para cima e valores negativos serão invertidos no sprite e escalados a menos que o valor utilizado é exatamente -1 (nesse caso, o sprite é simplesmente invertido ao longo do eixo y sem escalonamento).
  • Máscara - O índice da máscara, por exemplo. O valor padrão é -1 (o que significa que a instância usará a máscara de colisão para a atualmente atribuído ao índice sprite ), mas se for definido para qualquer outro recurso sprite, será usada a máscara de colisão para essa instância, em vez do sprite atribuído ao índice sprite.
  • Fricção - O atrito para usar cada frame do jogo para retardar a instância. Este valor será simplesmente deduzido do atual valor da velocidade de cada quadro de jogo (ou adicionado se o valor da velocidade for negativo).
  • Gravidade - A gravidade aplicará uma força constante na direção da gravidade (ver abaixo) da instância, influenciando sua velocidade (o padrão é 0). Note que a gravidade é uma força cumulativa e irá acelerar o object se você optar por não limitar a velocidade final.
  • Direção da gravidade - Pode ser usado para definir a direção do movimento quando a gravidade é maior que 0 (padrão é 270). Note que as direções no GameMaker são calculadas como 0° sendo a direita, 90° sendo para cima, 180° sendo para a esquerda e 270° sendo para baixo.
  • Pontuação - Defina a variável score.
  • Vidas - Defina a variável lives.
  • Saúde - Defina a variável health.

Uma vez selecionada a variável embutida a ser usada, você então define o valor para aquela variável. O valor a ser usado dependerá da variável selecionada (veja a lista acima para padrões e faixas) e você pode flag o valor a ser relativo à variável atual valor para a variável. Você também pode adicionar variáveis adicionais para definir na mesma ação, clicando no ícone de mais ao lado de a ação, e selecionando outra variável de instância e atribuindo-lhe um valor.
