Ordem do evento

Ao considerar os eventos no GameMaker, deve-se observar que a ordem exata de que TODOS os eventos vão ocorrer em cada etapa não pode ser claramente declarada, simplesmente porque depende do funcionamento interno do GameMaker e isto está sujeito a mudanças à medida que o software se desenvolve. Entretanto, existem certos eventos que sempre ocorrerão na mesma ordem.

O primeiro conjunto de eventos que sempre acontecerá da mesma forma são aqueles que ocorrem quando um room é inserido pela primeira vez. A ordem que os diferentes eventos irão disparar é:

  • Variáveis de Objeto / Variáveis de Instância são inicializadas (estas são as variáveis que você pode definir na IDE do Editor de Objeto ou do Editor de Sala). Se você tiver um object com variáveis anexadas, elas serão criadas antes de qualquer outra coisa e então quaisquer instâncias que tenham variáveis anexadas serão definidas depois para que você possa sobrescrever as variáveis object com variáveis específicas da instância na sala editor.
  • Criar Evento seguido do Código de Criação de Instância de cada instância - de modo que cada instância seja criada, ela executará primeiro seu evento Criar e depois seu Código de Criação de Instância antes de passar para a próxima instância a ser criada
  • Game Start Event - este evento é acionado uma vez no primeiro room do jogo para todas as instâncias colocadas no room do editor room (note que chamar game_restart() irá acionar este evento novamente.
  • Código de Criação de Salas - este é o código único escrito no Editor Room para quando um room é inserido pela primeira vez
  • Evento Room Start de todas as instâncias - um dos "outros" eventos da categoria "outros" e irá disparar para todas as instâncias, persistente ou não

Também vale a pena notar que você também pode definir a ordem na qual instâncias específicas são criadas dentro do próprio editor room movendo-as para cima ou para baixo na lista da janela de propriedades da camada de instância.

Além desses eventos específicos, a única ordem conhecida que sempre ocorrerá da mesma forma, não importa o que pertença aos eventos das três etapas e os diferentes eventos de sorteio . Estes sempre permanecerão consistentes, portanto, se você tem um código que depende de um tempo específico durante cada etapa de seu jogo, você deve usar:

  • Iniciar Evento Passo a Passo
  • Step Event (observe que o step event é executado um pouco antes das instâncias serem colocadas em suas novas posições)
  • Evento da etapa final

Todos os eventos para desenho também são sempre tratados na mesma ordem (exceto o evento Window Resize , que é acionado de forma diferente):

  • Evento de Pré-Sorteio
  • Evento de início do sorteio
  • Evento de sorteio
  • Evento de fim de sorteio
  • Evento pós-sorteio
  • Sorteio GUI Evento Inicial
  • Sorteio GUI Evento
  • Sorteio GUI Evento Final
