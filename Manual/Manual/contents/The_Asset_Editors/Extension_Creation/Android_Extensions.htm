Extensões do Android

Para criar um extension para Android você tem que fazer isso em duas partes. A primeira parte seria adicionar o próprio extension, junto com os arquivos necessários, e a segunda é criar as funções e macros/constantes que o extension requer.

As funções e constantes são adicionadas usando arquivos de placeholder para agrupá-las, assim você adicionaria um placeholder e então definiria as funções e macros como explicado na seção anterior. Para adicionar o resto dos arquivos, embora fosse necessário primeiro marcar a caixa de seleção Android na seção Plataformas extras do editor, que abrirá as propriedades da extensão Android:

Aqui você pode dar os seguintes detalhes:

  • Nome da classe: Seu extension pode ter múltiplas classes, com cada classe tendo suas próprias funções e constantes, portanto, você deve dar-lhe um nome que reflita seu propósito.
  • Permissões do Android: Aqui você pode adicionar quaisquer permissões extras que seu extension exigir. O que são essas permissões dependerá inteiramente do uso que o extension tem, e portanto você deve verificar a documentação fornecida pelo Google para a plataforma Android, ou, se você estiver usando um SDK de terceiros, a documentação que vem com o SDK. Para adicionar uma nova permissão, você precisa clicar no botão para adicionar uma permissão de espaço reservado, e então fazer um duplo clique nele para editá-lo ao que for necessário. Você pode remover as permissões usando o botão .
  • Injeção de código: Aqui você pode adicionar qualquer código a ser injetado (adicionado) em sua aplicação Android quando seu jogo for construído para teste ou lançamento final. Certifique-se de revisar isto (e suas permissões) cuidadosamente antes de enviar qualquer jogo para a loja, pois configurações incorretas farão com que seu jogo seja reprovado para envio. Consulte a seção abaixo para mais informações.

Injeção de código

Qualquer código inserido na janela "Injeção de código" precisará ser separado em grupos XML distintos, como mostrado na captura de tela acima. A etiqueta XML que você usa para um grupo determinará onde o código é injetado em sua aplicação.

Você pode usar a seguinte sintaxe para criar grupos de injeção de código:

<YYTagName>

// Code to inject

</YYTagName>

As seguintes etiquetas estão disponíveis para a plataforma Android:

NOTA Estes paths são apenas para VM; para YYC, o código injetado irá para o diretório {RUNTIME}\yyc\, onde o paths pode ou não ser equivalente aos de VM.

Arquivos fonte

Uma vez configurado corretamente, você precisará adicionar os arquivos necessários para que seu pacote extension funcione. Para fazer isso, você precisa clicar nos botões na parte inferior, ou Adicionar SDK ou Adicionar Fonte e então navegar até os arquivos que você deseja adicionar. Os arquivos adicionados serão armazenados no diretório AndroidSource junto com seu extension. Você pode abrir este local a qualquer momento clicando com o botão direito do mouse em no extension e selecionando Abrir no Explorer.

Para informações sobre sua criação nativa extensions para Android, veja Exemplos de fontes.
