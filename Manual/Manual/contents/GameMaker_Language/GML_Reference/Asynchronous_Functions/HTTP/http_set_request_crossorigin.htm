http_set_request_crossorigin

Com esta função você pode definir o tipo de origem cruzada a ser usada ao carregar imagens de um arquivo. A função é exclusivamente para a plataforma HTML5 e só é útil quando se carrega sprites a partir de um arquivo usando a função sprite_add(). Note que quando definida como "use-credentials" você não precisa mais dar um path absoluto ao sprite sendo carregado, mas em vez disso daria um path relativo (como mostrado no exemplo abaixo). Por padrão, o tipo de origem cruzada é definido como "anônimo".

Sintaxe:

http_set_request_crossorigin(origin_type);

ArgumentoTipo Descrição
origin_type The cross origin type to use (a string)

Devoluções:

Exemplo:

if string_lower(http_get_request_crossorigin()) != "use-credentials"
{
    http_set_request_crossorigin("use-credentials");
}
sprite_add("sprites/player_outfit_1.png", 0, false, false, 0, 0);

O código acima verificará primeiro o tipo de origem cruzada definido atualmente e se não for "credenciais de uso", então é definido para "credenciais de uso" e então um sprite é adicionado a partir de um arquivo.
