get_string_async

Esta função abre uma janela e exibe uma mensagem, bem como um espaço para o usuário inserir um string (que conterá o padrão fornecido string para começar). Esta é uma função assíncrona, e como tal o GameMaker não bloqueia o dispositivo em que está sendo executado enquanto espera por uma resposta, mas sim continua a executar eventos como de costume. Uma vez que o usuário tenha digitado seu string e pressionado o botão "Ok", um evento de diálogo assíncrono é acionado, o qual, somente para a duração desse evento, terá um mapa DS armazenado na variável async_load.

Este mapa conterá as três teclas, "id", "status" e "result". "id" é o valor que foi retornado pela função quando chamada, o "status" será ou true para o botão "Ok" sendo pressionado, ou false se a mensagem foi cancelada (onde aplicável, pois nem todas as plataformas de destino permitem que a mensagem seja cancelada). Finalmente, o "resultado" retornará o string que o usuário digitou (ou um string "" vazio, se nenhum foi fornecido).

NOTA Esta função é apenas para depuração e testee não deve ser usada em jogos lançados. Para isso, você deve criar uma interface de usuário personalizada para receber informações dos jogadores, para que você tenha controle total sobre a aparência e o comportamento dos diálogos.

Sintaxe:

get_string_async(string, default);

Argumento Tipo Descrição
string Cordão The message to show in the dialog.
default Cordão The default string.

Devoluções:

Async Request ID

Exemplo ampliado:

O evento de pressão do mouse esquerdo (por exemplo) do object que está mostrando a mensagem teria o seguinte código:

msg = get_string_async("What's your name?","Anon");

O acima mostrará uma mensagem solicitando que o usuário digite um string e pressione "Ok". A identificação da função é armazenada na variável "msg" e será usada no evento de Diálogos Assíncronos, como mostrado abaixo:

var i_d = ds_map_find_value(async_load, "id");
if i_d == msg
{
    if ds_map_find_value(async_load, "status")
    {
        if ds_map_find_value(async_load, "result") != ""
        {
            global.Name = ds_map_find_value(async_load, "result");
        }
    }
}

O código acima verifica a chave "id" do retorno DS Map em relação ao valor armazenado na variável "msg". Se forem os mesmos, então verifica se "Ok" foi pressionado (ao invés da janela ser fechada/cancelada) e se retorna true, então verifica o "resultado" de um string para ter certeza de que nenhum strings vazio foi retornado antes de definir uma variável global.
