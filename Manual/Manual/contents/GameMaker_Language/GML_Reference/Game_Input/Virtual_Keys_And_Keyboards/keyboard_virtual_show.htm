teclado_virtual_show

Esta função pode ser usada para mostrar o teclado virtual no dispositivo que executa o jogo. Quando você chama esta função, você precisa fornecer uma das seguintes constantes para cada um dos três primeiros argumentos:

  • Tipo de teclado: Esta constante é usada para definir qual conjunto de teclas estará disponível no teclado virtual. Os tipos de teclado disponíveis são:
Teclado Virtual Tipo Constante
Tipo de teclado Descrição
kbv_type_default O tipo de teclado padrão para o sistema atual.
kbv_type_ascii Um teclado somente em ASCII.
kbv_type_url Um teclado normal otimizado para entrada de URL. Normalmente apresenta um ".com" ou outras teclas de domínio, bem como as teclas "/" e ".".
kbv_type_email Um teclado normal otimizado para a entrada de e-mails. Normalmente apresenta caracteres "@" e ".".
kbv_type_numbers Um teclado só de números, geralmente exibido como um teclado numérico.
kbv_type_phone Um teclado de teclado telefônico. Normalmente só números - apenas com as teclas "*" e "#".
kbv_type_phone_name Um teclado otimizado para digitar tanto um número de telefone quanto um nome. Normalmente semelhante a um teclado ASCII, mas com uma seleção limitada de caracteres especiais.

  • Tipo de retorno: Esta constante é usada para definir o que é mostrado na tecla de retorno/ação do teclado virtual. Os tipos de retorno disponíveis são:
Teclado Virtual Tipo de Retorno Constante
Tipo de retorno Descrição
kbv_returnkey_default O título padrão da chave de retorno para o sistema atual.
kbv_returnkey_go Define o título da chave de retorno para "Ir".
kbv_returnkey_google Define o título da chave de retorno para "Google", ou para um ícone de busca genérica em alguns casos.
kbv_returnkey_join Define o título da chave de retorno para "Ir".
kbv_returnkey_next Define o título da chave de retorno para "Próximo".
kbv_returnkey_route Define o título da chave de retorno para "Rota".
kbv_returnkey_search Define o título da chave de retorno para "Pesquisar", ou para um ícone de pesquisa genérica em alguns casos.
kbv_returnkey_send Define o título da chave de retorno para "Enviar".
kbv_returnkey_yahoo Define o título da chave de retorno para "Yahoo", ou para um ícone de busca genérica em alguns casos.
kbv_returnkey_done Define o título da chave de retorno como "Feito".
kbv_returnkey_continue Define o título da chave de retorno para "Continuar".
kbv_returnkey_emergency Define o título da chave de retorno para "Chamada de Emergência".

  • Tipo de autocapitalização: Esta constante é usada para determinar como as palavras digitadas através do teclado virtual devem ser autocapitalizadas. Os tipos de autocapitalização disponíveis são:
Teclado Virtual Autocapitalização Tipo Constante
Tipo de autocapitalização Descrição
kbv_autocapitalize_none A autocapitalização está desativada.
kbv_autocapitalize_words As palavras serão auto-capitalizadas.
kbv_autocapitalize_sentences As sentenças serão auto-capitalizadas.
kbv_autocapitalize_characters Todos os caracteres serão capitalizados.

O último argumento é ativar/desativar o texto preditivo, e isso seria definido para true para permitir isso, e false caso contrário, mas note que só porque é permitido, não significa que será usado, pois isso dependerá das preferências do usuário em relação ao dispositivo. Quando em modo texto preditivo, o teclado virtual não irá gerar eventos normais de pressionamento de teclas do GameMaker. Ao invés disso, ele atualizará apenas o último caractere pressionado e as variáveis do teclado string. Isto se deve à incapacidade de detectar se uma mudança no campo de texto interno usado para detectar pressionamentos de teclas veio de uma tecla de teclado virtual real, ou de uma sugestão de texto. Nesses casos, você gostaria de ler a entrada keyboard_string ao invés de ler qualquer tipo de entrada de chave bruta.

É importante notar também que o usuário receberá teclados diferentes com diferentes capacidades dependendo do SO da plataforma, com as seguintes advertências para uso por alvo:

  • UWP: A implementação UWP não suporta return_key_type, autocapitalization_type ou predictive_text_enabled. A passagem destes parâmetros para uma função não causará nenhum erro, eles simplesmente não terão nenhum efeito sobre o teclado.

  • Android: Em Android, o tipo de teclado kbv_type_phone_name não é suportado e exibirá um teclado ASCII, e o tipo de chave de retorno só pode ser kbv_returnkey_go, kbv_returnkey_search, kbv_returnkey_next, kbv_returnkey_send ou kbv_returnkey_done. Se outros tipos de chave de retorno forem usados naquela plataforma, o sistema irá padrão para as chaves de retorno de substituição listadas abaixo:
Chave de retorno não disponível Chave de substituição
kbv_returnkey_google
kbv_returnkey_yahoo		 kbv_returnkey_search
kbv_returnkey_join
kbv_returnkey_route
kbv_returnkey_emergency		 kbv_returnkey_go
kbv_returnkey_continue kbv_returnkey_next

  • AndroidTV / FireTV: Os tipos de teclas de retorno personalizadas não são suportados nos teclados ASCII - a tecla de retorno padrão será sempre exibida.

  • tvOS: texto preditivo e auto-capitalização não são suportados em tvOS, e a altura do teclado retornado por eventos do sistema e a função keyboard_virtual_height() será padrão para a altura da tela devido ao fato de que ela abrange toda a tela e devido às funções de dimensão reta do teclado estarem desabilitadas em tvOS. Observe também que os eventos físicos (bluetooth) do teclado não serão transmitidos, a menos que o teclado virtual tenha sido aberto.

Chamando esta função, será gerado um Evento Assíncrono do Sistema, no qual o mapa DS async_load será preenchido com os seguintes pares chave/valor:

  • "event_type" - este será o "status do teclado virtual" quando acionado pelas funções do teclado virtual.

  • "screen_height" - a altura do teclado virtual (em pixels). Esta será 0 se o teclado for invisível.

  • "keyboard_status" - o status atual do teclado, retornado como uma das seguintes seqüências:
    • "escondido".
    • "oculto".
    • "mostrando"
    • "visível".

Sintaxe

keyboard_virtual_show(keyboard_type, return_key_type, autocapitalization_type, predictive_text_enabled);

Argumento Tipo Descrição
keyboard_type Teclado Virtual Tipo Constante Determines the keyset available on the virtual keyboard (see tables at the top).
return_key_type Teclado Virtual Tipo de Retorno Constante Determines what is shown on the return/action key of the virtual keyboard (see tables at the top).
autocapitalization_type Teclado Virtual Autocapitalização Tipo Constante Determines how/if the words typed via the virtual keyboard will be autocapitalized (see tables at the top).
predictive_text_enabled Booleano Set to true/false para habilitar/desabilitar a entrada de texto preditivo.

Devoluções:

N/A

Exemplo:

if input == false
{
    input = true;
    keyboard_virtual_show(kbv_type_numbers, kbv_returnkey_default, kbv_autocapitalize_none, false);
}

O código acima trará o teclado virtual do SO se a variável dada não estiver definida para true.
