aplicação_surface_enable

Com esta função você pode habilitar ou desabilitar a superfície de aplicação. Por padrão, a superfície de aplicação é habilitada e todos os desenhos para os eventos de Desenho serão feitos para isso e a superfície será desenhada para a tela. Entretanto, em alguns dispositivos mais antigos e chipsets específicos, isto pode resultar em um desempenho mais fraco. Nesses casos, você pode switch fora do desenho para a superfície de aplicação e todo o desenho dos eventos de Desenho será então feito diretamente para a tela buffer.

Você pode usar esta função para ativar ou desativar a superfície de aplicação a qualquer momento em seu jogo, mas você deve observar que ela requer pelo menos um quadro (etapa) para que as mudanças tenham efeito. Desativando-a irá switch fora de todas as correções de relação de aspectocomo definido nas Opções de Jogo para a plataforma alvo. Isto significa que você deve ter algum método no lugar que dimensione o jogo para o dispositivo, janela ou dimensão de tela necessários. Se você switch voltar a usar novamente, então as Opções de Jogo terão efeito novamente no próximo passo.

Recomenda-se usar esta função com cuidado, pois ela terá impacto na forma como tudo é atraído para a tela. Em geral, você gostaria de tê-la como opção em um menu de seu jogo, ou chamá-la apenas uma vez no início de seu jogo e escrever seu código, quer você a tenha habilitada ou desabilitada.

Sintaxe:

application_surface_enable(enable);

ArgumentoTipo Descrição
enable Set to enable (true) ou desabilitar (false) a superfície de aplicação.

Devoluções:

Exemplo:

if keyboard_check_pressed(vk_space)
{
    if application_surface_is_enabled()
    {
        application_surface_enable(false);
    }
    else
    {
        application_surface_enable(true);
    }
}

A verificação do código acima para um pressionamento de tecla e a alternância da superfície de aplicação, dependendo de seu estado (como em um menu de opções).
