file_find_first

Esta função devolverá o nome do primeiro arquivo que satisfaça a máscara e os atributos. Se tal arquivo não existir, então um string vazio será devolvido.

A máscara pode conter um path com wildcards, por exemplo C:\temp*.doc.

Os atributos dão os arquivos adicionais que você deseja ver, portanto os arquivos normais são sempre devolvidos quando satisfazem a máscara. Você pode somar as seguintes constantes para ver o tipo de arquivos que você deseja (usando |). Se você não desejar adicionar nenhum atributo, use 0.

Arquivo Atributo Constante
Constante Descrição
fa_readonly Arquivos somente leitura
fa_hidden Arquivos ocultos
fa_sysfile Arquivos do sistema
fa_volumeid Arquivos volumétricos
fa_directory Diretórios
fa_archive Arquivos arquivados

Os atributos só podem ser utilizados em Windows. Para todas as outras plataformas, use 0.

AVISO Esta função pode não funcionar como você espera devido ao GameMaker ser lixado! Consulte a seção sobre o Sistema de Arquivos para maiores informações.

NOTA: Esta função não funcionará de forma alguma no alvo HTML5.

Sintaxe:

file_find_first(mask, attr);

Argumento Tipo Descrição
mask Cordão The mask to use for searching.
attr Arquivo Atributo Constante(s) The specific file attribute to look for.

Devoluções:

String

Exemplo:

if directory_exists("\User Content")
{
    fileA = file_find_first("/User Content/*.doc", fa_readonly);
    fileB = file_find_next();
    fileC = file_find_next();
    file_find_close();
}

Este código verifica se o diretório especificado existe e, se existir, vai lá e retorna o primeiro arquivo "somente leitura" .doc encontrado. Em seguida, ele procura por mais dois arquivos e fecha o localizador de arquivos.

Você pode procurar por qualquer número de arquivos usando um while loop:

var files = [];
var file_name = file_find_first("/User Content/*.doc", fa_readonly);

while (file_name != "")
{
    array_push(files, file_name);

    file_name = file_find_next();
}

file_find_close();

O código acima cria uma matriz vazia para armazenar todos os nomes de arquivos que foram encontrados, e começa a procurar somente os arquivos .doc lidos. Se esse nome de arquivo não for um string vazio, ele o empurra para a matriz files, e então tenta encontrar o próximo arquivo. O loop continuará repetindo até encontrar um string vazio (o que significa que não há mais arquivos correspondentes), que é quando ele termina o loop e fecha o localizador de arquivos no final.
