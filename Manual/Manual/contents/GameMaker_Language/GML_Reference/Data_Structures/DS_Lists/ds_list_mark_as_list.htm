ds_list_as_list_as_list

Esta função irá "marcar" (ou "sinalizar") uma determinada posição dentro de uma lista de DS previamente criada como detentora de outra lista de DS. Esta funcionalidade é necessária ao codificar o JSON strings (que você pode criar usando json_encode()), mas também pode ser útil quando se aninham estruturas de dados, pois os itens marcados desta forma serão automaticamente coletados (destruídos) quando a lista DS pai for destruída. Isto significa que você não precisa passar manualmente pelo conteúdo da lista e destruir as estruturas de dados marcadas individualmente antes de destruir a lista "pai". Entretanto, se você apagar a posição da lista individualmente, a lista DS nela contida não será coletada e você deve chamar a função apropriada de destruição da lista DS antes de apagar o item da posição da lista pai. Observe também que se você chamar a função ds_list_clear() em uma lista, quaisquer itens assinalados como listas serão destruídos também quando a lista for limpa.

Vale notar também que se a posição que foi marcada mudar dentro da lista, a "marca" se moverá com ela, então se você tiver marcado a posição 3 na lista (por exemplo) e depois inserir mais 2 itens abaixo dela para que ela se mova para a posição 5, ela ainda será marcada como uma lista.

Sintaxe:

ds_list_mark_as_list(id, pos);

ArgumentoTipo Descrição
id The id of the list to mark.
pos The position within the list to mark.

Devoluções:

DS List ID or -1 (if it fails)

Exemplo:

var j_list = ds_list_create();
var sub_list = ds_list_create();
ds_list_add(sub_list, health);
ds_list_add(sub_list, lives);
ds_list_add(sub_list, score);
ds_list_add(j_list, sub_list);
ds_list_mark_as_list(j_list, 0);

O código acima cria duas listas de DS, em seguida, popula uma com vários valores de variáveis globais. Esta lista é então adicionada à segunda lista, e a posição "marcada" como tal, para que possa ser codificada corretamente mais tarde.
