ds_set_precision

Ao comparar valores, por exemplo, ao pesquisar em um mapa ou ordenar uma lista, o GameMaker deve decidir quando dois valores são iguais. Para strings e valores inteiros isto é claro, mas para números reais, devido a erros de arredondamento de ponto flutuante, números aparentemente iguais podem facilmente se tornar desiguais. Por exemplo, é possível que (5 / 3) * 3 não seja igual a 5! Para ajudar a evitar isto, um valor de precisão é usado em todas as funções de números reais, e quando a diferença entre dois números é menor do que esta precisão, eles são considerados iguais. A precisão padrão de 0,0000001 é usada para todas as funções de estrutura de dados, a menos que esta função seja alterada.

NOTA: Esta precisão é utilizada em todas as estruturas de dados, mas não em outras comparações em GML!

Sintaxe:

ds_set_precision(prec);

Argumento Tipo Descrição
prec Real The precision value (default 0.0000001)

Devoluções:

N/A

Exemplo:

ds_set_precision(0.0001);

O código acima mudará a configuração padrão de precisão para todas as funções da estrutura de dados.
