debug_get_callstack

Esta função gera um array de strings como o "calltack" onde o atual script é listado primeiro, e depois todos os outros scripts que foram executados para que o atual script fosse executado. O formato exato string variará dependendo da plataforma escolhida, mas na maioria das vezes terá o nome do script/evento, depois dois pontos : e o número da linha, semelhante a este:

"gml_Script_script2:1"
"gml_Script_script1:32"
"gml_Script_script0:64"
"gml_Object_object0_Create_0:1"

A função permite que um argumento opcional seja passado, que é a profundidade máxima do calltack retornado. Este valor é o número de scripts que estão incluídos na matriz retornada, a partir do atual script. Se este argumento não for especificado, então o Calltack completo será retornado.

Note que a matriz retornada conterá sempre 0 em sua última posição, após a listagem do indicativo scripts.

Sintaxe:

debug_get_callstack([maxdepth])

Argumento Tipo Descrição
[maxdepth] Real OPTIONAL A profundidade máxima do calltack que é devolvido

Devoluções:

Array

Exemplo:

if debug_mode
{
    if keyboard_check(vk_escape)
    {
        var _a = debug_get_callstack(4);
        for (var i = 0; i < array_length_id(_a); ++i;)
        {
            show_debug_message(_a[i]);
        }
    }
}

O código acima verifica se o modo de depuração está ativado e, se estiver, verifica se uma chave está sendo mantida pressionada. Nesse caso, ele emite a pilha de chamadas atuais para o console, com uma profundidade máxima de 4.
