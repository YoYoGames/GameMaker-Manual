sprite_merge

Esta função fundirá a sprite indexada no argumento 1 ("ind2") com a indexada no argumento 0 ("ind1"). As imagens em si NÃO são fundidas entre si, mas os índices de imagem são fundidas, com as sub imagens de sprite "ind2" anexadas às de sprite "ind1", ou seja: elas são adicionadas no final. Observe que, se o sprites for de tamanhos diferentes, então o anexo sprites é esticado para se ajustar ao tamanho da imagem para "ind1".

Esta mudança é permanente, e desde o momento em que você usar esta função até o fechamento do jogo ou o sprite apagado, o sprite que está sendo fundido será alterado, porém o sprite com o qual está sendo fundido permanecerá o mesmo.

NOTA: Nem a imagem asset que está sendo fundida, nem a imagem asset que é para fundir, pode ser um dos recursos originais do jogo. Você deve usar sprite_duplicate() primeiro para criar dois novos recursos (se você estiver fundindo inclusive o jogo assets) e depois fundir esses recursos.

NOTA: Esta função só é útil para o bitmap sprites e não funcionará com SWF ou JSON (Spine) sprites.

Sintaxe:

sprite_merge(ind1, ind2);

ArgumentoTipo Descrição
ind1 The index of the sprite to merge.
ind2 The index of the sprite that ind1 is to be merged with.

Devoluções

Exemplo:

var tspr;
tspr[0] = spr_add(working_directory + "Sprite Assets\explode1.png", 16, true, true, 0, 0);
tspr[1] = sprite_duplicate(spr_Explosion);
sprite_merge(tspr[0], tspr[1]);
sprite_index = t_spr[0];
sprite_delete(tspr[1]);

O código acima carrega um sprite em uma variável local e depois o funde com outro sprite que foi duplicado. Finalmente o sprite fundido é atribuído à instância sprite_index enquanto o sprite carregado é removido. para liberar a memória que utiliza. Observe que em algum momento após esta operação, como nos eventos Destroy ou Room End, você também precisará usar sprite_delete() para remover o duende fundido de memória também.
