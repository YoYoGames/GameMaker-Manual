sprite_create_from_surface

Com esta função você pode criar um sprite a partir de uma superfície previamente inicializada (o valor do índice de superfície é retornado quando você cria a superfície usando surface_create()). As coordenadas x e y que você então introduzir na função devem ser relativas à posição (0,0) da superfície (o canto superior esquerdo) e não à janela do jogo (nem à vista se você tiver uma ativa). Os argumentos de largura e altura estão em pixels e definem a largura e altura da parte da superfície a ser utilizada.

Definindo o argumento "removeback" para true removerá uma cor do sprite, verificando o pixel inferior esquerdo do sprite para a cor lá e depois usando-a como a cor a ser removida.

Se você escolher a opção "remover", você também pode querer que o GameMaker suavize as bordas do sprite, definindo o argumento "suavizar" para true. Tudo o que isso faz é criar uma borda semi-transparente ao redor das bordas do sprite após ter tido seu fundo removido.

Finalmente, você também pode especificar a origem x e y para o sprite. Este é o ponto onde o sprite é "fixo" na instância que o utiliza, e é sempre calculado como relativo ao canto superior esquerdo 0,0 de uma subimagem sprite. Assim, por exemplo, um sprite que é 32 x 32 pixels com estes valores definidos para (16,16) terá sua origem no centro.

Por padrão, todos os novos sprites têm suas caixas delimitadoras calculadas automaticamente (a caixa exata dependerá do tamanho e da transparência do sprite), entretanto você pode querer personalizar isso, caso em que você também deve usar a função sprite_collision_mask().

NOTA Quando você criar um sprite no GameMaker com este método, você deve se lembrar de removê-lo novamente (com sprite_delete()) quando não for mais necessário, caso contrário existe o risco de um vazamento de memória que irá retardar e, eventualmente, quebrar seu jogo.

Sintaxe:

sprite_create_from_surface(index, x, y, w, h, removeback, smooth, xorig, yorig);

Argumento Tipo Descrição
index Identificação da superfície The index of the surface to create from.
x Real The x position to copy from.
y Real The y position to copy from.
w Real The width of the area to be copied (from the x position).
h Real The height of the area to be copied (from the y position).
removeback Booleano Indicates whether to make all pixels with the background colour (left-bottom pixel) transparent.
smooth Booleano Indicates whether to smooth the edges.
xorig Real Indicates the x position of the origin in the sprite.
yorig Real Indicates the y position of the origin in the sprite.

Devoluções

Sprite Asset

Exemplo:

var surf;
surf = surface_create(32, 32);
surface_set_target(surf);
draw_clear_alpha(c_black, 0);
draw_sprite(spr_Body, 0, 0, 0);
draw_sprite(spr_Clothes, 0, 0, 0);
draw_sprite(spr_Hair, 0, 0, 0);
spr_custom = sprite_create_from_surface(surf, 0, 0, 32, 32, true, true, 16, 16);
surface_reset_target();
surface_free(surf);

O código acima cria uma superfície e armazena seu índice na variável local "surf". Em seguida, ele visa essa superfície, limpa-a e desenha vários sprites em cima uns dos outros. Finalmente, ele cria um novo sprite a partir da imagem composta desenhada na superfície e atribui seu índice ao "spr_Custom" antes de liberar a memória utilizada pela superfície.
