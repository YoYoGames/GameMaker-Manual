image_index

Um sprite é composto de uma ou mais sub-imagens que podem fazer o sprite aparecer animado como switch de um para o outro, ou podem ser alternados em código para dar diferentes "estados", como um botão tem em janelas. Se o sprite for animado, então você pode obter o quadro atual da animação verificando a variável image_index, ou se você quiser mudar o estado de uma sprite estática, você pode selecionar uma nova subimagem definindo esta variável para a subimagem desejada do sprite. Por favor, note que para que as mudanças nesta variável sejam visíveis, a instância deve ter ou nenhum evento de sorteio (e assim o GameMaker irá desenhar por padrão o sprite) ou ser desenhada usando uma das funções de desenho como draw_self() ou draw_sprite_ext() (fornecendo a imagem_index no argumento apropriado).

Observe que a definição desta variável diretamente em um quadro não acionará nenhuma mensagem de difusão que possa estar presente nesse quadro.

Observe também que, ao utilizar animação esquelética sprites, você ainda pode obter e definir os valores image_index - veja a função skeleton_animation_get_duration() para exemplos de como fazer isso.

Sintaxe:

image_index;

Devoluções:

(single precision floating point value)

Exemplo:

if (image_speed > 0)
{
    if (image_index >= image_number - 1) instance_destroy();
}

O código acima verifica se o sprite está animando, e se está então verifica se a imagem_index atual está no último frame e, nesse caso, destrói a instância.
