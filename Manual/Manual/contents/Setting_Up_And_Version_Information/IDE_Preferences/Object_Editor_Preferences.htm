Preferências do editor de objeto

As Preferências do Object Editor são usadas para definir certas propriedades para a janela do Object Editor. A seção principal contém a seguinte opção:

  • Clique para abrir Método de evento: Esta opção controla se você precisa realizar um clique duplo ou um simples clique em um evento object para abri-lo. Por padrão, isto é definido para dar um clique duplo.
  • Eventos herdados alfa: Esta opção permite mudar o alfa com o qual qualquer evento hereditário será desenhado na lista de eventos do object. Por padrão, isto é definido como 0,5 (os valores devem estar entre 0 e 1).
  • Formato do rótulo do evento: Aqui você pode ativar ou desativar os diferentes textos possíveis que podem ser aplicados à etiqueta do evento na lista de eventos do editor object. Por padrão, a descrição do evento é ativada apenas.
  • Adicionar o conteúdo padrão do evento: Por padrão esta opção é ativada e tudo o que ela faz é adicionar um simples comentário no editor de código para todos os novos eventos, para lembrar que você pode adicionar seu código lá.
  • Conteúdo do evento padrão: Aqui você pode editar os comentários que são adicionados por padrão na janela de código para novos eventos.

A seguinte sub-categoria também existe para o editor Object:

Confirmar Diálogos

Esta seção trata dos diálogos que podem ser mostrados durante a utilização do Editor Object, com a seguinte opção disponível:

  • Resposta automática a GML Conversão visual para a linguagem Gamemaker: Você pode usar o botão direito do mouse dentro do GML Visual workspace para selecionar a opção de transformar todo GML Visual para aquele evento em código GML. A ação padrão aqui é mostrar uma mensagem de aviso perguntando se você deseja continuar, mas você pode definir isto como "Ok" e o diálogo não será mostrado antes da conversão.
  • Resposta automática para exceder o limite de física vertex : Esta opção pode ser usada para controlar a mensagem pop up que é mostrada quando você excede o número permitido vertex no editor de forma física. A ação padrão aqui é mostrar uma mensagem de aviso, mas você pode configurá-la para "Ok" e o diálogo não será mostrado.
  • Escolha automática na eliminação de eventos: Esta opção pode ser usada para controlar a mensagem pop up que é mostrada quando você exclui um evento na lista de eventos do editor Object. A ação padrão aqui é mostrar uma mensagem de aviso, mas você pode definir isto como "Ok" e o diálogo não será mostrado.
