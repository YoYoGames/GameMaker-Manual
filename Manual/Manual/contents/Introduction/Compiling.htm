Compilação

Compilar seu jogo pode significar uma de duas coisas: compilá-lo para testes, ou compilá-lo para criar uma final pacote executável para uma plataforma alvo específica. Esta página tem como objetivo explicar detalhadamente ambas as opções.

Compilação para testes

A compilação de seu jogo para teste pode ser feita simplesmente pressionando o botão Play no topo do IDE, que lançará o jogo para teste usando o alvo especificado. Você também pode executar o jogo no modo Debug, testando usando o botão Debug . Isto irá executar o jogo, mas também abrirá a Janela de Debug, onde você poderá monitorar como seu jogo funciona e ver quaisquer problemas (veja a seção sobre Debugging para mais informações).

Configurações de destino

Por padrão, o GameMaker será executado e depurado usando a Máquina Virtual (VM), que é mais ou menos o mesmo que rodar no SO de mesa sendo usado. Entretanto, o GameMaker é um mecanismo de plataforma cruzada e você pode testar, depurar e compilar pacotes executáveis de seus projetos em várias plataformas alvo diferentes (as plataformas exatas disponíveis dependerão dos detalhes de sua licença). Para alterar a plataforma alvo atual você pode clicar em no botão Alvos para abrir a Janela de Alvos, que terá o seguinte aspecto (os detalhes exatos variam de acordo com seu tipo de licença):

Na parte superior, ao lado do botão Alvos, você tem as configurações atuais que indicam a plataforma e as configurações específicas que estão sendo usadas, e então o resto da janela é ocupada com os detalhes e opções para todos os alvos disponíveis que você pode selecionar para usar em seu lugar. Saída

Cada plataforma alvo pode ter um ou mais formatos de saída, sendo os principais:

  • VM: O alvo VM (Virtual Machine) usa um runner genérico para cada plataforma e depois interpreta o código para o seu jogo. Em geral esta opção é usada para testes devido a seus tempos de compilação mais rápidos, mas não oferece o mesmo aumento de desempenho que o uso da opção YYC (se disponível) oferece. Você pode usar isto para compilar pacotes executáveis para jogos menores ou jogos onde a performance nunca será um problema.
  • YYC: O YYC(YoYo Compiler) gera código C++ a partir de seu código GML, antes de usar o compilador C++ do alvo para compilá-lo em código nativo para a plataforma alvo. Ele elimina funções desnecessárias e executa uma série de outras técnicas de otimização para criar um pacote executável menor e com melhor desempenho.

    Isto pode aumentar o desempenho de seu jogo em pelo menos duas ou três vezes, especialmente em jogos de lógica pesada, e é ideal para aqueles projetos maiores ou com uso intensivo de CPU. Os tempos de compilação podem levar mais tempo e você deve sempre limpar o compilador cache antes de construir qualquer pacote final completo asset para uma plataforma alvo. Note que o alvo YYC pode exigir a instalação de ferramentas extras para a plataforma selecionada, caso contrário não funcionará - você pode encontrar mais informações sobre isto no Centro de Ajuda YoYo Games, bem como nas páginas de Preferências de alvos individuais detalhadas neste manual.
  • JavaScript: O alvo JavaScript só estará disponível para alvos específicos, como o alvo HTML5, e define o jogo a ser compilado em JavaScript puro. Isto usa ECMAScript 2015 (ou ES6) para o JS que ele produz.

Criando um pacote final executável

Visão geral

Você pode simplesmente clicar no botão Create Executable no IDE para iniciar a construção do compilador ou selecionar Create Executable a partir do Menu Build. Qualquer um dos processos iniciará o processo de construção que dependerá da plataforma alvo selecionada.

No alvo Opera GX, ele abrirá uma janela especial que lhe permitirá compilar e carregar seu jogo no GX.games; em todos os outros alvos, abrirá uma janela do explorador de arquivos onde você poderá dar o nome final que deseja usar para seu jogo executável, antes de clicar em Salvar para iniciar o processo de compilação e construção. Uma vez feito isso, os arquivos necessários serão gerados para que você possa distribuí-lo como desejar.

Para certas plataformas, a compilação de seu projeto de jogo exigirá que você tenha configurado o correto construir ferramentas (ver aqui) e também preencheu as Preferências de Plataforma apropriadas.

Dependendo de sua licença, você pode ter a opção de construir seu jogo via linha de comando, permitindo que você estabeleça uma integração contínua para seu projeto e racionalize seu processo de construção e teste. Consulte esta página para obter mais informações sobre a construção de linha de comando.

NOTA Antes de fazer uma construção final de seu projeto para lançamento, você deve sempre limpar o Asset Compiler Cache (usando o ícone "vassoura" na parte superior do IDE). O GameMaker irá cache muitos de seus arquivos de jogo para manter a velocidade de compilação a um mínimo e estes podem às vezes ficar corrompidos, portanto é mais seguro limpar esse cache antes de fazer uma compilação para o lançamento.

O tamanho máximo do pacote final do jogo é de 4GB, exceto para 32-bit Windows, onde é de 2GB. Veja dicas para reduzir o tamanho do jogo final.

Formatos Alvo

Cada opção alvo economiza para um formato específico da plataforma, listado abaixo:

  • Opera GX - Ao compilar para o alvo "Opera GX", você verá a janela "Opera GX Packaging" que primeiro lhe pedirá para entrar usando sua conta Opera:Note que você não precisará entrar aqui se você já tiver entrado no GameMaker usando sua conta Opera.

    Depois que você se inscrever, ele começará a compilar seu jogo e procederá ao seu upload para o GX.games. Você pode então clicar em "Edit Game on Opera" para editá-lo no GX.games DevCloud e publicá-lo:
    Consulte esta página para saber mais sobre o processo de upload do Opera GX.games, e esta página para configurar as várias exportações, como YYC e Mobile.
  • Windows - Compilando para o sistema operacional Windows geral, primeiro será solicitado que você escolha entre criar um instalador ou um pacote Zip , onde o instalador será um único executável que instalará seu jogo, e o arquivo Zip será um único arquivo comprimido no formato *.zip com todos os arquivos do jogo armazenados dentro dele (os arquivos precisarão ser extraídos para que o jogo seja executado). Se você marcar a caixa marcada como Remember Packaging Option, então o GameMaker se lembrará da escolha para todas as compilações futuras (isto pode ser redefinido ou alterado a partir das Preferências do Windows). Você pode saber mais no Centro de Ajuda do YoYo Games. Observe que o alvo Windows pode compilar executáveis de 32 bits ou 64 bits, dependendo do que você selecionou nas Opções Gerais do Jogo. Recomendamos fortemente que todos os novos projetos usem o x64 Runtime para criar executáveis de 64 bits, pois o runtime de 32 bits será depreciado e removido no futuro.
  • Ubuntu (Linux) - A compilação para Ubuntu lhe pedirá para escolher entre criar um arquivo .AppImage ou .zip, contendo um executável x64 ou um ARM (entretanto o AppImage não suporta ARM). Você pode saber mais no Centro de Ajuda do YoYo Games.
    .AppImage é usado para compartilhar como um pacote através de qualquer distribuidor, exceto Steam, enquanto .zip é usado somente para upload para Steam (pois usa o Steam Runtime especificamente). Por causa disso, o .zip criado não pode ser aberto como um arquivo zip regular.
  • HTML5 - Se você optou por construir HTML5, então um arquivo index.html (este é o nome padrão, mas você pode dar seu próprio nome também nas Preferências da Plataforma) juntamente com uma pasta contendo seus arquivos de jogo serão criados e salvos no local especificado. Para que seu jogo funcione, você precisará que ambos sejam carregados em um servidor. O index.html também pode ser personalizado para mostrar seu jogo com uma cor de fundo diferente, ou em uma posição diferente, etc... mas é necessário um conhecimento de HTML para isso, e você também pode especificar seu próprio arquivo de índice personalizado ao construir o pacote (veja as Opções de Jogo HTML5). Você pode saber mais no Centro de Ajuda do YoYo Games.
  • Android - Para dispositivos Android, você pode escolher construir um arquivo *.apk ou *.aab (Android App Bundle) da janela que é mostrada para salvar o jogo:

    O tipo de arquivo que você escolher dependerá da loja que você deseja atingir, com o arquivo *.aab sendo necessário para o Google Play, enquanto o arquivo *.apk pode ser usado em outras lojas. Você pode saber mais no Centro de Ajuda do YoYo Games.
  • iOS - Compilando para iOS será criado um arquivo xarchive que é então utilizado na Xtools para criar o pacote final iOS. Note que para compilar para iOS será necessário um computador Apple Mac rodando OSX ou superior, assim como os certificados e permissões relevantes. Você pode saber mais no Centro de Ajuda do YoYo Games.
  • macOS - A compilação para o alvo macOS irá primeiro solicitar que você escolha entre criar um DMG ou um arquivo Zip :
    "Pacote como DMG" criará um pacote instalador .dmg que pode ser usado para instalar o jogo em um computador Mac. Note que durante este processo de compilação você verá o instalador aparecer no Mac, que desaparecerá assim que o processo de compilação estiver completo.

    "Package as Zip" construirá um arquivo *.app ou um arquivo *.pkg, dependendo se você deseja ou não fazer o upload posterior para a loja Mac App Store. Como no iOS, você precisará de um computador Apple Mac rodando OSX ou superior, assim como os certificados e permissões relevantes. Você pode saber mais no Centro de Ajuda do YoYo Games.
  • Windows UWP - Para Windows UWP, o GameMaker criará um pacote *.appxbundle que poderá então ser carregado na Loja Windows ou distribuído em outro lugar. Ao clicar no botão Create Executable, você será solicitado a informar ao GameMaker qual tipo de pacote você gostaria de criar (pode ser para ARM, arquiteturas x64 e/ou x86), e você deverá escolher os alvos em que o jogo será suportado (você deve selecionar pelo menos um, mas pode selecionar os três ou qualquer combinação):

    Se você marcar a caixa marcada Package For Store Upload, então o pacote final criado será um arquivo *.appxupload, que é o que a Microsoft especifica que deve ser usado para enviar aplicativos para sua loja, como explicado neste artigo aqui (todas as arquiteturas escolhidas serão incluídas, assim como com o *.appxbundle). Você pode saber mais sobre a configuração e compilação para a plataforma UWP a partir do Centro de Ajuda do YoYo Games.

Uma vez criado seu pacote executável asset você pode então entregar o arquivo a outras pessoas ou colocá-lo em seu website para download, ou fazer o upload desses arquivos para os diferentes serviços de hospedagem para distribuição individual ou mesmo para lojas online (como Google Play, Apple App Store ou o Microsoft Store) para distribuição geral e varejo.

Observe que você é livre para distribuir os jogos que criar com o GameMaker da maneira que quiser, inclusive vendendo-os. Naturalmente, isto pressupõe que o sprites, imagens e sons que você usou para fazê-lo pode ser distribuído ou vendido também e que você tem os direitos legais para todos assets, e também pressupõe que o jogo está de acordo com o EULA da YoYo Games para GameMaker.
